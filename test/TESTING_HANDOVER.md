# Stray Dog — テスティング引継ぎ書

## プロジェクト概要

**Stray Dog** は、神保町・御茶ノ水エリアを舞台に、迷子の犬を探す位置情報連動型アドベンチャーゲームです。

- **技術スタック**: TypeScript, Vite, Leaflet (地図)
- **実行**: `npm run dev` → ブラウザでタップ操作
- **テスト**: `npm test`（vitest, 62件）
- **デプロイ**: `npm run build && surge dist/ straydog.surge.sh`

---

## 全体フロー

```
INTRO (タイプライター) → 4×4 フォトパズル → HUB (スポット一覧)
  → ストーリーテキスト → MAP (Leaflet + GPS)
  → スポット到着 → ミニゲーム (ぷよ/サイモン/クイズ)
  → バッジ獲得画面 → 次のスポット or HUB
  → 3バッジ収集 → 最終スポット解放 → エンディング
```

### 11. データ永続化

## テスト方法

### ユニットテスト
```bash
npm test           # 62 tests 全部流す
npx vitest --ui    # UIモード（要 @vitest/ui）
```

### 手動テスト（ブラウザ）
```bash
npm run dev        # http://localhost:5173
```

### スタンドアロンテスト
各ミニゲームは単体でテスト可能（`test/` ディレクトリ）:
```
test/puzzle4.html  # 写真パズル単体
test/puyo.html     # ぷよぷよ単体
test/simon.html    # サイモン単体
test/quiz4.html    # クイズ単体
```

### Hermes (E2E) への引継ぎ
E2Eテストフレームワーク（Playwright / Cypress）を使う場合の推奨シナリオ:
1. 画面遷移: intro → puzzle → hub の一連の流れ
2. 各画面の要素存在確認（IDベースのセレクタ）
3. localStorage の状態確認
4. モックGPSを使ったマップ遷移テスト
