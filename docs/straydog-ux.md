# Stray Dog — UX Design Spec

> スマホブラウザAR 犬探し謎解きのUI/UX設計

---

## 1. Design System

### 1.1 Color Palette

```
Gold           #FFD700   — 足跡、CTA、進行中ハイライト
DarkBg         #0A0A0F   — 背景（arg_app.html 踏襲）
Surface        #1A1A2E   — カード・モーダル背景
SurfaceLight   #2A2A3E   — ホバー・セカンダリ面
Border         #333      — 区切り線
TextPrimary    #E0E0E0   — 本文
TextSecondary  #888      — 補足テキスト
Success        #4CAF50   — クリア済み
Error          #C62828   — リトライ
CameraUI       rgba(0,0,0,0.4) — カメラ時のオーバーレイ床
PawGold        rgba(255,215,0,0.6) — 足跡の光
```

### 1.2 Typography

| 用途 | フォント | サイズ | ウェイト |
|------|---------|--------|---------|
| 見出し | Hiragino Sans / Noto Sans JP | 1.3rem | Bold |
| 本文 | 同上 | 0.9rem | Normal |
| 補足 | 同上 | 0.75rem | Light |
| 謎テキスト | 同上 | 1rem | Normal |
| 距離表示 | 同上 (mono) | 0.7rem | Light |
| カメラUIラベル | 同上 | 0.85rem | Bold |

### 1.3 Spacing Grid

```
4px  — アイコン内マージン
8px  — 要素間ギャップ
12px — カード内パディング
16px — 画面端マージン
24px — モーダル内パディング
32px — セクション間
```

### 1.4 Border Radius

```
8px   — カード・ボタン（小）
12px  — カード・モーダル（大）
16px  — モーダル
50%   — 円形要素
```

### 1.5 Shadows

```
Card        — 0 2px 8px rgba(0,0,0,0.3)
Modal       — 0 4px 24px rgba(0,0,0,0.5)
PawGlow     — 0 0 12px rgba(255,215,0,0.4)
ButtonGold  — 0 0 16px rgba(255,215,0,0.3)
SuccessGlow — 0 0 12px rgba(76,175,80,0.4)
```

### 1.6 Touch Targets

最小タップ領域: **44×44px** (iOS HIG / Material Design準拠)

**UI要素**:
- 地図は画面の64%高さ (calc(100dvh - 220px))

- 駅マーカー: クリア=緑, 未到達=金, 現在ターゲット=金＋点滅

- タイル: CartoDB dark_all (arg_app.htmlのOSMより視認性高い)

```
  border-radius: 50%;
  box-shadow: 0 0 0 4px rgba(255,215,0,0.3);
  animation: pulse 2s infinite;
```

**GPS非許可時**:
- 地図をデフォルトビューで表示（ユーザー位置なし）
- 下部に「📍 位置情報を許可すると現在地が表示されます」
- 駅はタップで選択可能 → 説明テキストのみ表示

### 2.3 CameraView / カメラAR画面

**状態**: 駅に到着 + カメラ起動中

```
┌──────────────────────────────┐
│ ← 戻る              📷      │
│ ┌────────────────────────┐   │
│ │                        │   │
│ │  📷 カメラ映像(全画面) │   │
│ │                        │   │
│ │   🐾 🐾 🐾            │   │
│ │   金色の足跡が         │   │
│ │   地面に浮かぶ         │   │
│ │                        │   │
│ │    ┌──────────┐       │   │
│ │    │ 📸 撮る  │       │   │
│ │    └──────────┘       │   │
│ │                        │   │
│ └────────────────────────┘   │
│                              │
│ 📍 さくら通り / あと35m     │
│                              │
└──────────────────────────────┘
```

**レイアウト**:
- カメラ映像（video）: position: fixed, 全画面, object-fit: cover
- オーバーレイ: position: absolute, 全画面重ね
- 下部インジケータ: linear-gradient(transparent, rgba(0,0,0,0.7)) で視認性確保

**ARレイヤー要素** (z-index stack):

```
z-index: 10 → UIボタン類（戻る、切替、フラッシュ）
z-index: 5  → AR Canvas（足跡、シルエット、エフェクト）
z-index: 1  → video（カメラ映像）
z-index: 0  → 背景黒
```

**画面要素**:

| 要素 | 位置 | 説明 |
|------|------|------|
| 戻るボタン | 左上 (16,16) | アイコン「←」44×44 |
| カメラ切替 | 右上 (16,16) | front/back切替アイコン |
| シャッター | 下部中央 | 直径72pxの円形ボタン、金色リング |
| 距離表示 | 下部左 | 次の駅までの距離 |
| 駅名表示 | 下部中央 | 現在の駅名 |
| ターゲットガイド | 中央 | 半透明シルエット（撮影対象の輪郭） |
| 足跡AR | 地面位置 | Canvasで動的に描画 |

**シャッターボタン**:

```
┌─────────────────────┐
│                     │
│         ┌───────┐  │
│         │  📸   │  │ ← 外円: Φ72, border: 3px solid #FFD700
│         │       │  │    内円: Φ56, background: white
│         └───────┘  │    押下時: scale(0.9) + shadow glow
│                     │
│   さくら通り         │
│   川面の光を撮影    │
└─────────────────────┘
```

**撮影後フィードバック**:
- シャッター音（Web Audio API で再生）
- 画面が一瞬白くフラッシュ（0.1s opacity 1 → 0）
- プレビューが1.5秒表示 → 自動でResultへ

### 3.2 Bottom Sheet (モバイル最適化)

MapViewの下部カードは draggable bottom sheet にする:

```
状態1: collapsed (64px高)
  ┌──────────────────────────────┐
  │ 🟡 次の駅まで 120m    📷    │
  └──────────────────────────────┘

状態2: expanded (画面40%)
  ┌──────────────────────────────┐
  │ 🟡 Station 3: 神田川         │
  │ 📍 さくら通り                │
  │ 📏 あと120m (徒歩2分)        │
  │                              │
  │ 川面の光を撮影して、         │
  │ 犬が残した手がかりを         │
  │ 見つけ出そう                 │
  │                              │
  │ ┌────────────────────────┐   │
  │ │    📷 カメラを起動     │   │
  │ └────────────────────────┘   │
  └──────────────────────────────┘
```

**実装**: touch-start/touch-move/touch-end でスワイプ検出
- CSS: `transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1)`

## 4. Micro-Interactions

| 要素 | トリガー | アニメーション | 時間 |
|------|---------|---------------|------|
| 足跡AR | 常時 (カメラ時) | 出現→0.5s保持→フェード→次の位置へ移動 | loop 2s |
| 足跡 (地図上) | 常時 | 点滅 (opacity 0.4↔1) | 1.5s |
| シャッターボタン押下 | tap | scale(0.9) → 0.1s delay → scale(1) | 0.2s |
| シャッターフラッシュ | 撮影時 | white overlay opacity 0→1→0 | 0.15s |
| バッジ獲得 | clear時 | scale(0)→1.2→1 + glow | 0.5s |
| ストーリーテキスト | clear時 | 1文字ずつ appear (typing) | 2-4s |
| 駅完了チェック | clear時 | scale(0)→1 rotate(0→360) | 0.4s |
| ページ遷移 | 画面切替 | slide-left/right | 0.3s |
| シルエット表示 | 駅到着時 | fade-in + subtle float | 0.5s |
| コンフェッティ | ゴール時 | 30粒が random fall | 3s loop |
| ホーム進捗bar | 更新時 | 該当ドットが fill アニメ | 0.3s |

**足跡ARアニメーション詳細** (カメラ時):

```
各足跡パターン:
  1. Canvas上のランダム位置に paw-print を描画
  2. 出現時: scale(0)→1 + opacity 0→0.6 (0.3s)
  3. 滞在: opacity 0.6 で1.2s 浮遊 (y軸 -2px 上下)
  4. 消滅: opacity 0.6→0 + scale(1)→0.8 (0.5s)
  5. 次の位置に移動: x += random(10,30), y += random(-5,5)
  6. 5-7個の足跡が異なるタイミングでループ

色: rgba(255,215,0,0.6) + blur 2px
サイズ: 24×24px (CSSでいうところの)
```

## 6. Error & Edge Case States

| 状況 | UI | 挙動 |
|------|----|------|
| カメラ許可拒否 | 赤枠メッセージ + 「QRで進む」button | QRモードにフォールバック |
| GPS許可拒否 | 地図固定 + 手動駅選択UI | 駅をタップで選択 → 説明のみ |
| GPS精度低 (±50m以上) | 「📍 精度が低いです」warning | 目安として表示、ブロックはしない |
| カメラ非対応ブラウザ | 「📷 非対応」+ 代替テキスト | 「写真を選ぶ」input[file] で代用 |
| localStorage満杯 | エラー通知 + 進行度失われるwarning | 古いセッションをクリアする確認 |
| オフライン | キャッシュ表示 + 「オフラインです」 | キャッシュ済み駅までは動作 |
| バッテリー節約モード | アニメーション簡略化 | prefers-reduced-motion 検出 |
| ダークモード固定 | 常にダークテーマ（変更不可） | OS設定に関わらず dark 固定 |

## 7. Accessibility

| 要件 | 対応 |
|------|------|
| コントラスト比 | 本文 4.5:1以上 (金色#FFD700 は large textのみ) |
| フォーカス表示 | ボタンに outline: 2px solid #FFD700 |
| 代替テキスト | 写真キャプションは aria-label |
| 画面回転 | 常にポートレート固定 (iOSは制限不可だが推奨表記) |
| prefers-reduced-motion | アニメーションをすべて無効化 |
| 音声操作 | シャッターボタンに aria-lable="写真を撮る" |
