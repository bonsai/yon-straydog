# Stray Dog — 仕様書・シナリオ進行

> **プロジェクト**: 神保町エリアの位置情報アドベンチャーゲーム
> **対象**: モバイルブラウザ（PWA未対応、要ネット接続）
> **スタック**: TypeScript + Vite 6 + Leaflet + Vitest

---

## 1. 全体フロー

```
起動
  │
  ├─ [INTRO] ステップテキスト (Adventure Overlay)
  │   テキストが1ブロックずつ表示される
  │   ブロック終了後「ゲームをするかやめるか？」の はい/いいえ
  │   はい → 4x4パズルへ
  │   いいえ → 終了（画面そのまま）
  │
  ├─ [4×4 フォトパズル] 16枚のタイルを入れ替えて犬の写真を復元
  │   クリア → HUBへ
  │
  ├─ [HUB] 4つのスポットカード
  │   各カードをタップ → そのスポットのストーリーテキスト
  │   条件を満たすとロック解除
  │
  ├─ [MAP] Leaflet地図 + GPS (デバッグ時はモックモード)
  │   スポットに接近 (50m以内) → 到着
  │   「謎を解く」 → ミニゲーム起動
  │
  ├─ [MINI-GAME] スポットごとのミニゲーム
  │   クリア → バッジ獲得画面
  │
  ├─ [次のスポット / HUBに戻る]
  │   3バッジ集める → 最終スポット解放
  │
  └─ [COMPLETE] エンディング、もう一度遊ぶ
```

## 3. アドベンチャーシステム

### 3.1 Step キュー

`game-state.ts` のキュー方式。1テキスト = 1Step。

```
interface Step {
  type: 'text' | 'action'
  text?: string           // type='text' で表示
  action?: () => void     // type='action' で実行
  nextPhase?: GamePhase   // キュー消費後の遷移先フェーズ
}
```

### 3.2 テキスト表示の流れ

1. `buildIntroSteps()` / `buildStorySteps()` / `buildHintSteps()` でStep配列を生成
2. `setSteps(steps)` → `startAdventure()` で表示開始
3. 各テキストStepは `#adventure-overlay` に表示
4. テキストブロック末尾で「ゲームをするかやめるか？」の はい/いいえ
   - **はい** → `goNext()` で次のStepへ
   - **いいえ** → 何もしない（そのテキストに留まる）
5. 最後のaction Stepで自動的に次のフェーズへ遷移（パズル/マップ等）

### 3.3 テキストブロックの定義

**イントロ** (`buildIntroSteps`):
```
壁のQRコードを読み取ると、そこには ひと組の夫妻の姿があった。
「あの…すみません。うちの犬がいなくなってしまったんです。」
「妻が妊娠中で、動けなくて…どうか…」
あなたは夫妻の代わりに、犬を探すことにした——
夫妻は一枚の写真を差し出した。だが——写真は砕け散っている。
元にもどせばなにかわかるかもしれない。
```

**スポットヒント** (`buildHintSteps`):
```
📌 {スポット名} に向かおう
{ヒント文}
```

**ストーリー** (`buildStorySteps`):
- 各スポットの幕テキスト（`STORY_SCENES` から）

## 4. GPS / マップシステム

### 4.1 通常モード

- `navigator.geolocation.watchPosition()` でリアルGPS追跡
- 許容誤差: `enableHighAccuracy: true`
- タイムアウト: 10秒
- 到着判定: スポットから **50m以内**

### 4.2 モック/デバッグモード（現在はデフォルト）

- 8秒以内にGPS応答がない場合 → 自動でモックモードに移行
- モックモードでは「📍 次のスポットへ」ボタンでスポット間を移動
- デバッグ確認のための `isMockMode` フラグ

### 4.3 犬マーカー

- 未訪問スポット間を4秒おきにランダム移動（ただの演出）
- `L.divIcon` で 🐕 絵文字表示

### 5.1 ぷよぷよ (`src/game/puyo-game.ts`)

- 6×12 グリッド
- 7色（🍓🍊🍋🍈💎🥛🍇）
- 目標: 指定色のぷよを **3グループ** 消去
- 操作: ← → ↑ ↓ Space
- クリア: 自動クローズ → `completeCurrentSpot()`

### 5.2 シモンセズ (`src/game/simon-game.ts`)

- 4パッド（ドレミソ）、2×2グリッド
- 4ラウンド
- 音声: Web Audio API 三角波
- クリア: ラウンド4クリア → 自動クローズ

### 5.3 四字熟語クイズ (`src/game/quiz4-game.ts`)

- 48問のプールからランダム出題
- 1文字入力、Enter/ボタンで回答
- 正解 → クリア、不正解 → リトライ

## 8. デバッグ / テスト方法

### GPSモックモード
- GPS非対応環境または8秒タイムアウトで自動起動
- 地図画面の「📍 次のスポットへ」ボタンでスポット間移動
- 制限: `indexedDB` 不使用、`localStorage` のみ

### テスト用スクリプト
- 各ミニゲームは `test/` ディレクトリの独立HTMLでもテスト可能
- 単体テスト: `npm run test` (62 tests, Vitest)
- 型チェック: `npx tsc --noEmit`
- ビルド: `npm run build`

### 既知の課題
1. 実機GPSの調整（未デバッグ）
2. モバイルタッチ最適化（特にぷよぷよ操作）
3. BGM/SE未実装（シモンのみWeb Audio）
4. PWA未対応（manifest, service worker）
