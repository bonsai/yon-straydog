# Stray Dog — コード解析

## 状態管理

```js
appState: "intro",
completed: JSON.parse(localStorage.getItem("sd_completed") ?? "[]"),
introDone: localStorage.getItem("sd_intro_done") === "true",
currentSpot: null,
userPos: null
```

### LocalStorage キー

| キー | 用途 |
|------|------|
| `sd_completed` | クリア済みスポット一覧 |
| `sd_intro_done` | オープニング既読 |
| `sd_4x4_done` | 写真パズル完了 |

## 4×4パズル

- **サイズ**: 4×4 = 16ピース
- **画像**: `0.jpeg` を16分割
- **関数**:

| 関数 | 処理 |
|------|------|
| `createPuzzleState()` | 状態生成（シャッフル/非シャッフル） |
| `shuffleTiles()` | タイルシャッフル |
| `selectOrSwap()` | タップ選択・交換 |
| `isSolved()` | 完成判定 |

## 既知の課題

1. **LocalStorage 改ざんで全クリア可能**
   ```js
   localStorage.setItem("sd_completed", '["s0","s1","s2","s3"]')
   ```
   チート防止は設計範囲外（デバッグモードで最初から直接起動可能）

2. **座標がフロントに完全公開**
   公開スポット情報のため問題なし（ゲーム成立に必須）

3. **ストーリー全文がクライアントにある**
   ゼロインストールWebアプリの特性上不可避
