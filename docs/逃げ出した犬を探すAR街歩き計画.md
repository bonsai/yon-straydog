# 逃げ出した犬を探す — AR街歩き謎解き計画

## コンセプト

さぼうるで進行していたら、突然犬が逃げ出した。スマホのカメラで街の風景を撮影すると、ARで隠された足跡・手がかり・謎が浮かび上がる。カメラ越しの世界にだけ見える「もう一つの神保町」を辿り、犬を探す旅。

- **メカニクス**: カメラ撮影 → 画像認識/位置情報でトリガー → ARオーバーレイで謎表示
- **デバイス**: スマホブラウザ（WebAR / WebXR / 8th Wall 不使用、素のJS + CSS 3D + カメラAPI）
- **ストーリー**: 逃げ出した犬「ゴールド」（ゴールデンレトリバー）を探す

---

## ストーリー詳細

### 導入

あなたは神保町の老舗喫茶「さぼうる」で、友達とクリームソーダを飲んでいる。そこへ1匹のゴールデンレトリバー「ゴールド」が店内に入り込む。店主に怒られて、犬は慌てて外に飛び出した。

「あの犬…さっきまでここにいたはずなのに、もう見えない」

スマホのカメラを起動すると、**カメラ越しにだけ見える金色の足跡**が道に浮かんでいる。あなたはその足跡を追いかけ始める。

### ストーリーの層

| 層 | 内容 |
|----|------|
| **表の謎** | 犬の足跡を追う → 各所で犬が残した「物」（拾ったもの、こぼしたもの、落とし物）を見つける |
| **裏の謎** | 犬を通じて、神保町の「失われた記憶」がよみがえる → 過去の人々の人生の断片が見える |
| **最終章** | 犬を追うことで、実は自分自身の「失くした何か」を見つける物語 |

### 犬「ゴールド」の正体（オプション／深掘り設定）

- 正体: 過去の時間から迷い込んだ記憶の犬
- 「さぼうる」から逃げ出した理由: 店内のアンティーク時計に反応した（時計に閉じ込められた誰かの記憶を運んでいる）
- 探すことで、神保町に眠る様々な時代の人々の記憶がほどける

## コアメカニクス

### 1. カメラ撮影トリガー

各スポットで**スマホのカメラ**を起動し、特定の対象を撮影すると謎が進行する。

| トリガー種別 | 実装方式 | 例 |
|-------------|---------|-----|
| **QR/マーカー** | jsQR でコード読取 | 店先のポスター、マンホールのシール |
| **位置情報＋時間** | GPS + 時刻判定 | 特定座標で夕方だけ現れる手がかり |
| **画像マッチング** | カメラ画像の特徴点比較 | 建物の形状、看板の色、街灯の形 |
| **光/影の検出** | 輝度センサー＋方角 | 影の向きが示す方向 |

### 2. ARオーバーレイ表示

カメラプレビュー上に、以下のAR要素をCSS 3D/Canvasで重ねる：

- 🐾 **足跡**: 地面に浮かぶ金色の足跡（進行方向を示す）
- 💬 **犬の思考**: 犬が残した「📸写真で残したいもの」がモノローグで表示
- 🔍 **発見物**: 犬が落としたもの（鍵・骨・ボール・写真の切れ端）
- 👻 **記憶の残像**: 過去の人々の姿がシルエットで現れる

### 3. 謎の種類

| 種類 | 内容 | 例 |
|------|------|-----|
| **写真謎** | 特定のものを撮影 → 謎が解ける | 「金色のものを3つ撮影せよ」 |
| **比較謎** | 2つの写真を並べると差分に謎 | 現在と過去の同じ場所の写真を重ねる |
| **アングル謎** | 特定のアングルで撮影 | ビルの隙間から見える特定の形 |
| **集め謎** | 複数の写真を決められた順に撮る | 虹色順に7つのものを撮影 |
| **影謎** | 影の形を撮影 | 特定時刻の影が示す矢印の方向 |

### エリア

神保町周辺（さぼうるを起点として半径800m圏内）

### 全7ステーション + ゴール

| # | 場所 | トリガー | 謎内容 | 所要 |
|---|------|---------|--------|------|
| 0 | **さぼうる**（スタート） | 店内のアンティーク時計を撮影 | 犬が逃げ出した。カメラ越しに金色の足跡が現れる | 5分 |
| 1 | **神保町交差点〜靖国通り** | 信号機の色が変わった瞬間を撮影 | 足跡が交差点で二手に分かれる – 右か左か？ | 10分 |
| 2 | **古書店街（ほるぷを中心に）** | 古書店のショーウィンドウに映った影を撮影 | 犬が本を1冊くわえている。その本のタイトルは？ | 15分 |
| 3 | **さくら通り（神田川方面）** | 川面の光を撮影 | 水面に映る犬の姿 – しかし上を向くと何もいない | 10分 |

## 写真判定ロジック（MVP案）

MVPでは、厳密な画像認識ではなく**GPS＋簡易的なトリガー**で進行させる。

| 方式 | 精度 | 実装難度 |
|------|------|---------|
| GPS圏内 + カメラ起動 → 自動進行 | ★★★ (低) | 低 |
| GPS圏内 + QR読取 → 謎開放 | ★★ (低〜中) | 低 |
| GPS圏内 + 特定オブジェクト撮影 (簡易色検出) | ★★★★ (中) | 中 |
| 画像マッチング (TensorFlow.js) | ★★★★★ (高) | 高 |

**推奨（MVP）**: GPS圏内に入ったらカメラUIを表示。画面上に写すべき対象のシルエットガイドが表示され、ユーザーが撮影ボタンを押すと自動進行。画像認識は後段で追加。

## 技術スタック（初期案）

| 要素 | 技術 | 備考 |
|------|------|------|
| カメラ起動 | `navigator.mediaDevices.getUserMedia` | ユーザータップ必須。`facingMode: 'environment'` 指定 |
| 位置情報 | `navigator.geolocation.watchPosition` | 既存 `arg_app.html` と同じ方式 |
| ARオーバーレイ | Canvas 2D + CSS 3D transform | 8th Wall等の外部SDK不使用 |
| QR読み取り | `jsQR` (CDN) | 軽量、単体で動作 |
| 簡易画像認識 | ColorThief + 輝度解析 | 色相・輝度で簡易マッチング |
| データ保存 | localStorage | オフライン対応 |
| オーディオ | Web Audio API / SpeechSynthesis | 足跡の音、犬の鳴き声など |
| PWA対応 | manifest.json + Service Worker | ホーム画面に追加可能に |

## 懸念・注意点

- **プライバシー**: カメラ・位置情報の許諾フローを丁寧に
- **バッテリー**: GPS + カメラ常時起動は消費大 → 間欠動作 or ユーザー操作時のみ起動
- **ブラウザ互換性**: Chrome(Android)とSafari(iOS)の挙動差に対応必須（下記詳細）
- **実店舗協力**: さぼうる・YON・その他店舗の許可が必要な場合あり
- **雨天**: カメラを屋外で使う想定 → 傘をさしながらでも操作しやすいUI

## ブラウザ互換性対応表

| 機能 | Chrome (Android) | Safari (iOS) | 対策 |
|------|-----------------|--------------|------|
| `getUserMedia` | ○ 問題なし | △ HTTPS必須・ユーザータップ必須 | 初回タップでカメラ開始。HTTP不可 |
| `facingMode: 'environment'` | ○ 認識する | ○ 認識する | リアカメラ指定を明示 |
| フルスクリーンAPI | ○ `Element.requestFullscreen` | △ `webkitEnterFullscreen`（video要素のみ） | videoタグのplaysinline+webkit-playsinline属性必須 |
| 画面回転 | ○ `orientationchange` | ○ 同じ | `screen.orientation.lock` はiOSで未対応のため諦める |
| Web Speech API | ○ 問題なし | △ 音量が小さい・声質限定的 | 代替としてテキスト表示を併用 |
| AudioContext | ○ | △ ユーザー操作後に再開必要 | `resume()` をタップイベント内で呼ぶ |
| localStorage | ○ | ○ 問題なし | — |
| Service Worker (PWA) | ○ フル対応 | △ 限定的・Safari特有の制約あり | PWAは後回し・まずは通常ブラウザで動作 |
| `input type="file" capture` | ○ | ○ 問題なし | カメラ起動のフォールバックに使える |
| Canvas 2D | ○ | ○ 問題なし | ARオーバーレイはCanvasで統一 |
| CSS 3D transform | ○ | ○ 問題なし | — |
| jsQR | ○ | ○ 問題なし | 純JSなので両対応 |
